# Contributing to Learning the Baseline

We welcome community contributions to scale this analog behavioral framework. Because this system relies on absolute physical constraints, all contributions must adhere strictly to the system rules outlined below.

## How You Can Contribute

### 1. Hardware Chassis Modding (Prototype Iterations)
* Submit alternative blueprint designs (e.g., 3D printing STL files, CNC routing specs) for the universal matte-slate casing.
* Requirements: The viewport must maintain the exact 1.50" H × 4.50" W mechanical cut at the lower 33.3% boundary coordinate.

### 2. Canvas Asset Submissions (/assets)
* Submit 52-week master card matrix assets.
* Format: `.png` format only.
* Resolution: Strict 1290 × 2796 pixels.
* Styling: `#000000` True-Black canvas background with pure `#FFFFFF` geometric monospaced text pinned between `y=1400` and `y=1850`.

## Submission Process
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/ChassisMod`).
3. Commit your changes and open a Pull Request explaining how your modification respects the Project Charter's constraints.
